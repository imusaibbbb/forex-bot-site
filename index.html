01 — Multi-timeframe stack

Four timeframes, one decision.

Bias is set on the higher timeframes, confirmed on the lower ones. No single chart gets a vote on its own — direction must agree from H4 down to the candle that actually triggers the order.

01
H4
Macro trend
EMA50 vs EMA200 establishes the dominant directional bias for the pair.
02
H1
Trend agreement
Must confirm the same bias as H4 — disagreement cancels the setup outright.
03
M15
Entry confirmation
RSI, volume, BOS, and support/resistance retest are all evaluated here.
04
M5
Precision entry
Final confirmation candle determines the exact bar the order is placed on.
02 — Entry rules

Every condition, both directions.

The buy and sell logic are exact mirrors of each other — no asymmetric shortcuts, no exceptions.

BUYlong bias
  • ✓EMA50 sits above EMA200 on H4 and H1
  • ✓Price trades above both EMAs
  • ✓RSI(14) reads between 50 and 70
  • ✓Bullish break of structure confirmed
  • ✓Current volume above rolling average
  • ✓Price retests a detected support zone
  • ✓Bullish confirmation candle on M5
SELLshort bias
  • ✓EMA50 sits below EMA200 on H4 and H1
  • ✓Price trades below both EMAs
  • ✓RSI(14) reads between 30 and 50
  • ✓Bearish break of structure confirmed
  • ✓Current volume above rolling average
  • ✓Price retests a detected resistance zone
  • ✓Bearish confirmation candle on M5
03 — Smart money concepts

Reading where liquidity actually sits.

Five structural detectors run on every confirmation-timeframe candle, feeding the structure score and gating entries to genuine institutional footprints rather than indicator crossovers alone.

BOS

Break of structure

Price closes beyond the last significant swing point, confirming the prevailing trend is continuing.

CHoCH

Change of character

The first break against the prevailing trend — an early reversal warning the scorer weighs against new entries.

FVG

Fair value gap

Imbalanced candles leave a gap that price tends to revisit before continuing — mapped on every scan.

OB

Order blocks

The last opposing candle before an impulsive move, marking where large positions were likely built.

LQ

Liquidity sweep

A wick beyond a swing high or low that closes back inside range — the classic stop-hunt signature.

S/R

Support & resistance

Swing highs and lows are clustered into zones, weighted by how many times price has reacted there.

05 — Trade management

What happens after the fill.

BREAKEVEN
  • →Stop moves to entry once the trade reaches +1.5R — the position can no longer lose money.

  • →Activates at +1R, then trails by a multiple of current ATR — giving winners room without giving back gains.

08 — Project layout

Modular by design.

Strategy, risk, execution, and backtesting are separate, independently testable modules — swap a broker, a scoring weight, or a timeframe without touching the rest.

forex_bot/ ├── main.py # entry point — backtest / paper / live ├── config/config.yaml # every setting lives here ├── core/ │ ├── strategy.py # multi-timeframe decision engine │ ├── scoring.py # the 5-factor confluence gate │ ├── smc.py # BOS / CHoCH / FVG / order blocks │ ├── risk_manager.py # sizing, SL/TP, trailing, breakeven │ ├── mt5_connector.py # live broker bridge + reconnection │ └── paper_broker.py # simulated fills for paper mode ├── backtest/backtester.py, optimizer.py ├── dashboard/dashboard.py └── tests/ # 24 unit tests, all green
09 — Get running

Backtest first. Always.

01

Install dependencies

Create a virtual environment and run pip install -r requirements.txt. MetaTrader5 itself is Windows-only — backtesting works anywhere.

02

Configure

Open config/config.yaml and set your broker login, symbols, and risk limits. Nothing else needs editing.

03

Backtest

Run python main.py --mode backtest against historical data and review win rate, drawdown, and the Monte Carlo spread.

04

Paper trade

Switch to --mode paper and let it run against live prices with simulated fills for at least several weeks.

05

Go live, deliberately

Only once paper results hold up, point --mode live at a demo server first, then a funded account — on a Windows VPS running as a service.

!

This is engineering, not a profit guarantee. Every rule here is real and tested — but markets change, and no backtest substitutes for your own paper-trading review on your own broker before risking capital.
